# Philosophy

WP Trend Watcher exists to help WordPress developers stay current without turning WordPress news into another full-time job.

The project is a workflow first and a tool second.

## Human Reviewed

AI may collect, summarize, and synthesize information.

Humans own:

- conclusions
- recommendations
- opinions
- final publishing decisions
- what is worth paying attention to

The project should make review easier, not remove it.

## Budget Conscious

The workflow should fit realistic personal usage.

Targets:

- less than $1/month for personal usage
- less than $5/month for heavier usage

The project should track cost in public reports whenever practical.

## Provider Agnostic

The project should avoid hard dependence on one AI provider.

Potential providers may include:

- local models
- OpenAI
- Anthropic
- Gemini
- other compatible APIs

Phase 1 should not overbuild provider support. Start with the smallest provider abstraction that allows future swapping.

## Open Source First

The workflow should be understandable by WordPress developers who do not care about AI agents or complex automation systems.

Public docs should explain:

- what the workflow does
- what it does not do
- how sources are selected
- how reports are reviewed
- what it costs to run

## Simple Before Clever

Avoid unnecessary architecture until the core workflow proves useful.

Phase 1 avoids:

- vector databases
- embeddings
- knowledge graphs
- agent swarms
- autonomous publishing
- dashboards
- historical trend tracking

## The Real Value

AI summaries are not the product.

The useful layer is human judgment applied to summarized ecosystem activity.
